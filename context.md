# context.md

## 1.1 — Product Definition

ytclfr is a YouTube video intelligence pipeline that accepts a single YouTube video URL as input and automatically extracts all structured, meaningful content from it — including spoken words (via speech-to-text), on-screen text (via OCR), and temporal context — then classifies the video type (recipe, movie list, script, book list, product list, etc.) and produces a structured JSON response containing the extracted items, timestamps, confidence scores, and provenance data. The system is used by developers who consume the JSON API directly, researchers who process large sets of YouTube videos programmatically, and general public users who submit a URL through a web UI and receive clean, readable structured content without watching the video.

## 1.2 — Frozen Tech Stack

Language (backend) → Python 3.11+ — mature ecosystem for ML/AI workloads, native async support, FastAPI compatibility
Language (frontend) → TypeScript — type safety for the public-facing UI, modern tooling
API framework → FastAPI — async-first, automatic OpenAPI docs, native Pydantic integration
Task queue / worker system → Celery 5 + Redis (broker) — battle-tested distributed task execution for Python
Database (primary) → PostgreSQL 16 — robust relational store, supports pgvector extension for vector search
Database (vector/semantic search) → pgvector (PostgreSQL extension) — vector similarity search without a separate service, fits single-machine deployment
Search / segment index → pgvector with GIN indexes — full-text and vector search within PostgreSQL, avoids a separate OpenSearch cluster in V1
Cache → Redis 7 — in-memory cache and Celery message broker in one service
Temporary media storage → Local filesystem path — confirmed for single-machine laptop deployment, no S3 dependency in V1
OCR engine → Tesseract 5 (via pytesseract) — open-source, widely supported, no API key required
ASR / transcript engine → faster-whisper (model=small, device=cpu) — confirmed decision, local CPU inference, no external API dependency
AI/LLM client → Ollama (llama3.1:8b, local) + Groq (llama-3.3-70b-versatile, cloud) — local model for routine extraction, cloud model for hard reasoning tasks
Schema validation → Pydantic v2 — native FastAPI integration, strict runtime validation, JSON Schema generation
Testing framework → pytest + httpx — async test client for FastAPI, fixtures, parametrize support
Linting / type checking → ruff + mypy — fast linting with ruff, strict type checking with mypy
CI system → GitHub Actions — integrated with repo hosting, free tier sufficient for V1

STACK FROZEN. No library may be added or swapped without a new entry in decisions.md and explicit human approval.

## 1.3 — V1 Scope Boundary

### IN SCOPE — V1:

- YouTube URL ingestion (single video URL per request)
- Video download to local temporary storage via yt-dlp
- Automatic temporary media cleanup after processing
- Video metadata extraction (title, duration, channel, thumbnail)
- Preflight router that classifies video content type (speech-heavy, list/edit, mixed)
- ASR transcription via faster-whisper with word-level timestamps
- OCR extraction of on-screen text via Tesseract with frame-level timestamps
- Temporal alignment layer merging ASR transcript and OCR text into a unified timeline
- LLM-powered structured extraction using Ollama (local) for routine tasks
- LLM-powered structured extraction using Groq (cloud) for hard reasoning tasks
- Content type detection and classification (recipe, movie_list, book_list, song_list, product_list, script, summary, mixed)
- Confidence scoring per extracted field
- Provenance tracking (timestamp in video where each item was found)
- Structured JSON output via FastAPI REST API
- Job submission endpoint (POST URL, receive job_id)
- Job status polling endpoint (GET job status by job_id)
- Job result retrieval endpoint (GET structured output by job_id)
- JWT authentication for API endpoints
- Asynchronous job processing via Celery workers
- Job persistence and status tracking in PostgreSQL
- Web UI for general public users (submit URL, view results)
- API documentation via OpenAPI (auto-generated by FastAPI)
- Basic rate limiting on job submission

### OUT OF SCOPE — V1 (do not build, do not stub, do not mention in code):

- Non-YouTube URL support (Vimeo, TikTok, Instagram, Dailymotion)
- Live stream processing
- Real-time video processing
- Video translation or subtitle translation
- Free-form Q&A or chatbot interaction about video content
- Content recommendation engine
- Plagiarism or copyright checking
- Batch URL submission (multiple videos in one request)
- User accounts with persistent history (beyond JWT auth)
- Video hosting or media serving
- Webhook notifications for job completion
- Multi-language translation of output
- Mobile native applications (iOS, Android)
- Video editing or clipping
- Audio-only podcast processing (YouTube videos only)
- Custom LLM model fine-tuning
- Admin dashboard
- Usage analytics or billing
- Multi-tenant architecture
- Distributed multi-machine deployment

## 1.4 — Authentication Strategy

Mechanism: JWT (JSON Web Tokens)

Endpoints requiring authentication:
- POST /api/v1/jobs (submit a new video processing job)
- GET /api/v1/jobs/{job_id} (check job status)
- GET /api/v1/jobs/{job_id}/result (retrieve structured output)

Endpoints NOT requiring authentication:
- GET /api/v1/health (health check)
- GET /docs (OpenAPI documentation)
- GET /openapi.json (OpenAPI schema)
- All static frontend assets

Credential storage:
- JWT_SECRET_KEY — environment variable, never committed to source control
- JWT_ALGORITHM — environment variable (default: HS256)
- JWT_EXPIRY_MINUTES — environment variable (default: 60)

## 1.6 — Environment Variables

### Database

DATABASE_URL — PostgreSQL connection string (postgresql://user:pass@host:port/dbname) — FastAPI, Celery Worker, Alembic
DATABASE_POOL_SIZE — Connection pool size — FastAPI

### Cache / Queue

REDIS_URL — Redis connection string (redis://host:port/db) — FastAPI, Celery Worker, Celery Beat

### Temporary Storage

TEMP_MEDIA_PATH — Absolute filesystem path for temporary video/audio downloads — Celery Worker
TEMP_MEDIA_MAX_AGE_SECONDS — Maximum age before temp files are auto-cleaned — Celery Worker

### LLM / AI

OLLAMA_BASE_URL — Ollama server URL (http://host:port) — Celery Worker
OLLAMA_MODEL — Ollama model name (llama3.1:8b) — Celery Worker
GROQ_API_KEY — Groq cloud API key — Celery Worker
GROQ_MODEL — Groq model name (llama-3.3-70b-versatile) — Celery Worker
LLM_REQUEST_TIMEOUT_SECONDS — Max wait time for LLM responses — Celery Worker
LLM_MAX_RETRIES — Max retry attempts for failed LLM calls — Celery Worker

### ASR

WHISPER_MODEL_SIZE — faster-whisper model size (small) — Celery Worker
WHISPER_DEVICE — Compute device for whisper (cpu) — Celery Worker
WHISPER_COMPUTE_TYPE — Compute type for whisper (int8) — Celery Worker

### OCR

TESSERACT_CMD_PATH — Path to Tesseract binary (if not in system PATH) — Celery Worker
OCR_FRAME_SAMPLE_RATE — Frames per second to sample for OCR — Celery Worker

### Authentication

JWT_SECRET_KEY — Secret key for signing JWT tokens — FastAPI
JWT_ALGORITHM — JWT signing algorithm (HS256) — FastAPI
JWT_EXPIRY_MINUTES — Token expiry duration in minutes — FastAPI

### Service

API_HOST — FastAPI bind host — FastAPI
API_PORT — FastAPI bind port — FastAPI
WORKER_CONCURRENCY — Number of concurrent Celery worker processes — Celery Worker
CELERY_TASK_TIME_LIMIT — Max seconds per Celery task — Celery Worker

### General

LOG_LEVEL — Logging level (DEBUG, INFO, WARNING, ERROR) — All components
ENVIRONMENT — Deployment environment name (development, staging, production) — All components

## 1.7 — Deployment Target

The system runs on a single laptop (bare metal): the FastAPI API server, Celery worker, PostgreSQL database, and Redis cache all run on the same machine; temporary media is stored on the local filesystem at the path specified by TEMP_MEDIA_PATH; this is a single-machine, non-distributed deployment suitable for development and personal use in V1.
